## 🎬 애니메이션 변화

### 페이지 로드

**Before:**
```
콘텐츠 즉시 표시
```

**After:**
```
1. 전체 페이지 페이드인 (0.6초)
2. 리스트 항목 순차 등장
   - 첫 번째: 0ms
   - 두 번째: 50ms
   - 세 번째: 100ms
   - ...
3. 위에서 아래로 부드럽게 슬라이드
```

### 호버 효과

**Before:**
```
배경색 살짝 밝아짐 (0.15초)
```

**After:**
```
버튼: 색상 반전 슬라이드 (0.3초)
카드: 위로 떠오름 + 그림자 생성 (0.3초)
입력: 하단 바 확장 + 위로 이동 (0.2초)
```

### 체크박스 토글

**Before:**
```
즉시 체크마크 표시
```

**After:**
```
1. 180도 회전 (0.2초)
2. 바운스 효과
3. 배경 노란색으로 채워짐
4. 체크마크 나타남
```

## 🎨 질감과 깊이

### 그레인 텍스처

**Before:**
```
완전히 평면, 디지털 느낌

```

**Before:**
```

box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
```

**After:**
```
Brutal 하드 섀도우
box-shadow: 
  4px 4px 0 #ffed4e,           ← 선명한 노란 그림자
  8px 8px 0 rgba(255,237,78,0.2); ← 부드러운 확장
  
→ 3D 레이어처럼 보임
→ 카드가 "떠있는" 느낌
```

## 📏 간격 시스템

### Before
```
padding: 16px, 24px (임의)

```

```
gap: 8px 배수로 통일
```

## 🎨 스타일 가이드

### DO (해야 할 것)

✅ 대문자 사용 (강렬함)
✅ 모노스페이스 폰트 (명확성)
✅ 각진 모서리 (brutalist)
✅ 네온 강조색 (시선 끌기)
✅ 의미 있는 애니메이션
✅ 3px+ 두꺼운 테두리
✅ 하드 섀도우 (4px 오프셋)
✅ 색상에 의미 부여

### DON'T (하지 말아야 할 것)

❌ 둥근 모서리 (너무 부드러움)
❌ 얇은 테두리 (약해 보임)
❌ 과도한 그라데이션
❌ 의미 없는 애니메이션
❌ 낮은 대비 색상
❌ 일반 산세리프 폰트
❌ 부드러운 섀도우
❌ 애니메이션 남발

## 🚀 성능 비교

### 렌더링 성능

**Before:**
```
FPS: 60 (안정)
Repaints: 보통
GPU 사용: 최소
```

**After:**

**Before:**
```

```

**After:**
```
CSS: ~7.5KB (커스텀 CSS)
Fonts: JetBrains/Space/Azeret (~180KB)
Total: ~187.5KB

→ CSS는 -37% 감소
→ 폰트는 약간 증가 (3개 사용)
→ 전체적으로 +15% (허용 범위)
```
